Step 1 — LLM Mode
Theoretical Design
Developer describes the new module. LLM designs against the original blueprint — as if the project was built perfectly. It reasons aloud, surfaces assumptions, and produces a theoretical design with explicit dependencies.
Developer Request ● Developer

2
Step 2 — Agent Mode
Gap Report
Agent reads the LLM's theoretical design and compares it against the actual drifted codebase. It identifies what the design assumes exists — but doesn't. Blockers vs warnings are clearly separated.

3
Step 3 — Developer Decision
Gap Decisions
Review each gap identified by the Agent. Decide: fix it now before Admin Dashboard build, or accept the risk and proceed. These decisions shape the final LLM design. You own the technical debt choices — not the AI.
Your Decisions Select fix or accept for each gap
4
Step 4 — LLM Final
Final Admin Dashboard Design
LLM redesigns the Admin Dashboard using three inputs: original blueprint (intent), agent's gap report (reality), and your gap decisions. The result is a design the agent can actually execute against what exists — not what was planned.
Final Admin Dashboard Design
5
Step 5 — Stage 3 Mandatory
Agent Execution Package
The complete handover. Part A remediates the blockers you chose to fix. Part B builds the Admin Dashboard. Agent executes A → B in sequence. Zero ambiguity, zero assumptions.
